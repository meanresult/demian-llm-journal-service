# Demian LLM Journal Service
## LLM 기반 성찰 일기 서비스

## 1. 서비스 개요

Demian은 사용자의 하루 기록을 단순 저장하는 것을 넘어,  
LLM을 활용하여 성찰을 유도하고 과거 기록과 연결해주는 저널링 서비스입니다.

기존 일기 앱은 단순 기록에 머무르는 경우가 많지만,  
Demian은 다음과 같은 차별점을 갖습니다.

- 기록 기반 맥락 이해
- 성찰을 유도하는 AI 응답
- 구조화된 데이터 설계
- 향후 회상 및 추천 기능 확장 가능

본 프로젝트는 이러한 서비스를 구현하기 위한  
백엔드 아키텍처와 데이터 구조 설계를 중심으로 개발되었습니다.

## 2. 서비스 흐름

1. 사용자가 하루 기록 작성
2. 기록이 구조화된 형태로 저장
3. LLM이 맥락 기반 질문 또는 피드백 생성
4. 과거 기록과 연결 가능하도록 설계


## 3. 기술적 목표

- 기록 데이터를 구조화하여 재현 가능한 형태로 저장
- LLM 연동을 서비스 로직과 분리하여 확장 가능 구조 설계
- Supabase(PostgreSQL) 기반 데이터 관리
- Docker를 활용한 실행 환경 재현성 확보


## 4. 기술 스택

- FastAPI
- Supabase (PostgreSQL)
- OpenAI / Claude
- Docker

## 6. 왜 이 프로젝트를 만들었는가

LLM을 활용한 서비스는 많지만,
대부분 단순 호출 수준에 머무르는 경우가 많습니다.

본 프로젝트는
LLM을 중심이 아닌 '도구'로 두고,
기록 데이터를 어떻게 구조화하고 관리할 것인가에 초점을 맞추었습니다.

이는 장기적으로
- 회상 기능
- 감정 분석
- 사용자 맞춤 질문 생성
- 추천 시스템
